# 技术架构文档

## 系统架构概述

ClearCut 采用现代 Web 技术栈，基于边缘计算架构设计，确保高性能、高可用性和良好的用户体验。

## 技术栈

### 前端层
- **框架**: Next.js 14 (App Router)
- **语言**: TypeScript 5.0+
- **样式**: Tailwind CSS + CSS Modules
- **状态管理**: Zustand + React Query
- **UI组件**: Shadcn/ui + Lucide React
- **表单处理**: React Hook Form
- **文件上传**: React Dropzone

### 边缘层
- **部署平台**: Cloudflare Pages
- **边缘函数**: Cloudflare Workers
- **对象存储**: Cloudflare R2
- **CDN加速**: Cloudflare CDN
- **KV存储**: Cloudflare KV (缓存)

### API层
- **图片处理**: Remove.bg API
- **API框架**: Next.js API Routes
- **速率限制**: Upstash Redis
- **用户认证**: Auth.js (NextAuth)

### 开发工具
- **代码规范**: ESLint + Prettier
- **测试框架**: Jest + React Testing Library
- **类型检查**: TypeScript
- **Git钩子**: Husky + lint-staged
- **监控告警**: Sentry
- **网站分析**: Plausible

## 数据流设计

### 图片处理流程
```
1. 用户上传图片
   ↓
2. 前端验证（格式、大小）
   ↓
3. 发送到API路由（FormData）
   ↓
4. 调用Remove.bg API
   ↓
5. 获取处理结果
   ↓
6. 存储到Cloudflare R2（24小时）
   ↓
7. 返回处理结果URL
   ↓
8. 前端显示预览
   ↓
9. 用户下载图片
```

### 错误处理流程
```
1. 捕获异常
   ↓
2. 记录错误（Sentry）
   ↓
3. 分类错误类型
   ↓
4. 返回友好错误信息
   ↓
5. 前端显示错误提示
   ↓
6. 提供解决方案
```

## 性能优化策略

### 1. 前端性能优化
- **代码分割**: 动态导入组件
- **图片优化**: 自动压缩和格式转换
- **缓存策略**: 浏览器缓存和CDN缓存
- **懒加载**: 图片和组件懒加载

### 2. 后端性能优化
- **边缘计算**: 全球分布式部署
- **API缓存**: 相同图片哈希值缓存结果
- **并发控制**: 智能任务队列管理
- **压缩传输**: Gzip/Brotli压缩

### 3. 成本优化
- **智能缓存**: 减少重复API调用
- **图片压缩**: 上传时自动压缩
- **使用限制**: 免费用户额度控制
- **监控告警**: 实时监控API使用成本

## 安全设计

### 1. 文件安全
- 图片格式验证（白名单）
- 文件大小限制（最大5MB）
- 病毒扫描（基础防护）
- 文件类型检测（Magic Bytes）

### 2. API安全
- API密钥保护（环境变量）
- 请求频率限制（IP/用户）
- 错误信息脱敏
- CORS配置（严格限制）

### 3. 数据安全
- 临时文件加密存储
- 自动数据清理（24小时）
- 隐私政策声明
- GDPR/CCPA合规

### 4. 网络安全
- HTTPS强制加密
- 安全头配置（CSP, HSTS等）
- DDoS防护（Cloudflare）
- 定期安全审计

## 技术决策理由

### 1. 选择Next.js的原因
- 服务端渲染提升SEO和首屏性能
- App Router提供更好的开发体验
- 内置API路由简化后端开发
- 优秀的TypeScript支持

### 2. 选择Cloudflare的原因
- 全球边缘网络加速
- 免费额度满足初期需求
- 一体化解决方案（Pages+Workers+R2）
- 优秀的安全防护能力

### 3. 选择Remove.bg的原因
- 行业领先的处理质量
- 稳定的API服务
- 合理的定价模型
- 良好的开发者支持
